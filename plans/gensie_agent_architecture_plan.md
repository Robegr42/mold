# GenSIE 2026: Final Agent Architecture Plan

## The Strategy for Small Model Survival
Small language models shatter when forced to balance rigid JSON constraints against complex zero-shot inference. The GenSIE 2026 challenge demands precise schema extraction from Spanish texts using offline models under 14B parameters, and penalizes hallucinations with merciless score reductions. To survive these constraints, we abandoned monolithic prompting. Our hybrid architecture extracts universal mechanics—TypeScript schema compression, extract-or-null gating, and dialect-awareness—and solders them directly into foundational utilities. By stripping these rote tasks from the inference context, we preserve the model's raw inferential heat for three surgical core agents. The TwoPassAgent drafts and refines raw extractions, the GroundedAgent anchors entities to verbatim text strings, and the AuditorAgent vetoes unverified claims before final serialization. This orchestration requires a stable substrate; we must first strip away the mechanical friction of schema enforcement by embedding it into the system's foundational invariants.

## Universal Invariants (The Foundation)
Raw token capacity dictates survival in zero-shot environments. Forcing a sub-14B parameter model to juggle Spanish regionalisms, navigate verbose JSON validation rules, and perform complex semantic extractions causes the model to hemorrhage hallucinations. We abandon the practice of loading these structural prerequisites into the main inference prompt. Instead, we offload these rote syntax-level guardrails into foundational architectural invariants—rules the system enforces automatically before the active agent ever sees the task.

Consider the physical bloat of a standard JSON schema. Feeding raw JSON syntax into a tight prompt window floods the context with nested brackets, string quotes, and structural noise. The `compress_schema_to_ts` utility in `utils/schema.py` intercepts this verbose payload and compiles it down into a dense TypeScript interface. We carve away the syntactic fat to deliver a compact, native representation of the target structure. The model spends its cycles analyzing the text, bypassing the friction of untangling schema definitions, preserving razor-thin bandwidth.

We implement the `InvariantPromptMixin` to stabilize the model's behavior. This class mixin acts as an unyielding substrate that injects two non-negotiable rules into every execution path. First, it enforces an absolute "Extract-or-Null" gate. When faced with an empty schema field, the model panics and invents a fact; the mixin chokes this instinct, forcing a null instead. Second, it provides "Dialect-Awareness" instructions. A Madrileño’s 'vosotros' clashes with a Chileno’s 'ustedes'—the mixin primes the extraction engine to navigate these linguistic minefields.

Embedding these invariants at the deepest layer of the architecture isolates the core reasoning tasks from mechanical overhead. Our higher-level agents inherit a clean, tightly controlled, sanitized sterile field for extraction where they can apply their full inferential weight toward surgical data extraction.

## Core Agent Architectures
By abstracting the structural mechanics into universal invariants, we clear the execution floor for three specialized agents. Each agent embodies a distinct behavioral persona designed to outmaneuver the inherent failure modes of small language models working under strict JSON constraints.

The TwoPassAgent solves the cognitive bottleneck of simultaneous reasoning and formatting by deliberately splitting the brain of the language model. Forcing a sub-14B parameter model to extract nuanced semantic meaning while fighting escaping quotes and bracket alignment severely degrades its inferential capacity. Pass one permits the model on the text—allowing it to map relationships and chart concepts using unstructured natural language. With the analytical heavy lifting complete, pass two acts as a mechanical compiler. It consumes the unconstrained analysis from the first pass and executes a rigid transcription into the target schema. This two-phase architecture insulates the reasoning process from the formatting syntax.

The GroundedAgent targets the persistent "hallucination by completion" failure—the model's desperate instinct to populate empty fields. This agent functions as an uncompromising detective demanding physical text evidence. Before it can assign a value to the final output schema, the GroundedAgent must extract the exact Spanish substring and lodge it within a mandatory `_source_quote` property. If the model cannot locate the precise source characters, it loses the authorization to write the target value. This forced anchoring binds every extraction strictly to the origin document, halting the model's ability to invent data just to satisfy an expected structure.

The AuditorAgent introduces internal friction by forcing the model to interrogate its own work through a hostile review cycle. The architecture triggers a multi-stage validation loop, replacing the standard single-shot approach. The system first compiles a preliminary JSON draft. The execution flow then shifts, effectively passing a red pen to the model and commanding it to operate as an adversarial inspector. The auditor sweeps the draft against the origin text to unearth fabricated entities. When it detects an unverified claim, the agent aggressively strikes the field; it replaces the hallucinated data with a hard `null`. This internal policing guarantees only text-supported extractions survive final serialization.

## The Sequence of Refactoring
The refactoring of `src/gensie/baseline.py` proceeds through a series of architectural cuts.

Dissecting the monolithic `baseline.py` with a digital scalpel begins with decoupling the schema compilation logic. We extract the `compress_schema_to_ts` function and isolate it within a dedicated `src/gensie/utils/schema.py` module. This cut clears the central namespace and establishes a stable dependency for our architectural components.

Scaffolding the `InvariantPromptMixin` follows. We encode the Extract-or-Null gate and Dialect-Awareness primitives directly into this foundational class. Wiring these constraints into the prompt generation pipeline guarantees every downstream inference request automatically inherits these absolute behavioral boundaries.

With the invariants bolted into place, we construct the specialized extraction engines. We implement the `TwoPassAgent`, `GroundedAgent`, and `AuditorAgent` classes, forcing each core unit to inherit from both the base `GenSIEAgent` and our new `InvariantPromptMixin`. This dual inheritance physically solders their unique operational tactics to our unyielding structural rules.

The final tactical shift reroutes the participant registry. We update the `OfficialParticipant` class to dismantle the brittle baseline configurations and register our three hardened agents into the execution flow. This final integration exposes our distinct extraction strategies to the evaluation framework.
